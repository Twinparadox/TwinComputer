1.사용 설명서.

1. 먼저 MySQL에 테이블을 만들어야 합니다.

2. CREATE TABLE table이름 ( id VARCHAR(8) NOT NULL PRIMARY KEY , kind VARCHAR(50) NOT NULL,
name VARCHAR(200) NOT NULL, price INT(11) UNSIGNED ) ;

3. 그런 다음, dbconn.inc 화일에서 아래 부분을 자신의 MySQL 에 맞게 수정합니다.
...
$dbconn = mysql_connect("localhost","jopc306 [MySQL 접속ID]","47083 [MySQL 접속 PASSWORD]") || die("데이터베이스 연결에 실패하였습니다.");
$status = mysql_select_db("joapc [MySQL DB 이름]");
...

4. modifyform.html 에서는 자신의 제품목록을 추가합니다.

5. assemble.html 에서는 DB 저장된 id 값을 <select> 태크에서 각 제품의 value 값과 일치하게 작성하시고 사용하시면 됩니다.

2. 각 화일 설명.

index.htm - 지금 보고 계신 이 화일 입니다.
postform.html - 제품을 입력하는 양식 화일입니다.
post.php - 제품입력양식의 값들을 처리하는 화일입니다.
list.html - 제품목록을 출력합니다.
modifyform.html - 제품목록에서 지정한 품목의 수정및 삭제 양식을 출력합니다.
modify.php - 제품수정양식에서 넘어온 값을 처리하는 화일입니다.
delete_check.htm - 제품수정양식에서 삭제하는 품록을 한 번 더 확인하는 화일입니다.
delete.php - 제품수정양식에서 넘어온 값을 삭제합니다.
assemble.html - PC 견적서 양식을 출력합니다.
assemble_process.php - PC 견적서에서 넘어온 값을 처리하고, 견적내용을 출력합니다.
footer.php - 하단 부분의 내용. (수정해서 사용하세요~)

3. 개선해야할 문제점.

1. 제품값을 출력하는 과정에서 숫자 세 자리마다 (,)를 삽입하여 사용자가 보기 편하게 해야 합니다.
2. 제품목록에서 제품종류별검색을 아직 구현하지 않았습니다.
3. 제품목록이 너무 아래로 길게 표시돼므로 수정해야합니다. 게시판처럼 아래 번호를 붙여서 이동하도록 하는 코드를 구현해야 함.
4. 제품추가시 제품종류를 선택하면 제품번호가 자동으로 할당되도록 하게 해야 함.
5. 제품목록의 내용을 자동으로 PC 견적서 양식(HTML 화일)로 생성하는 과정을 구현해야 함.
6. PC 견적결과 양식에서 원하는 품목만 수정하여 결과를 볼 수 있게 해야 함.
7. 기타 개선 사항은 버젼이 높아지면서 나아지리라고 봅니다. (문제는 항상 시간!입니다.)
